# تردد کارخانه بتن (Concrete Factory Entry/Exit App)

اپلیکیشن اندروید برای ثبت ورود و خروج **پرسنل**، **ماشین‌آلات**، **مراجعین و مهمانان** و **رانندگان** یک کارخانه بتن.

## ویژگی‌ها
- صفحه اصلی با ۴ دسته اصلی.
- برای پرسنل/ماشین‌آلات/رانندگان: ثبت‌نام یک‌بار در سیستم، سپس ورود/خروج نامحدود.
- برای مراجعین: هر ورود با پرسیدن «نام مهمان» و «واحد مورد مراجعه» ثبت می‌شود.
- **قانون منطقی**: تا وقتی خروج یک فرد ثبت نشود، امکان ثبت ورود مجدد او وجود ندارد (پیام خطا نمایش داده می‌شود).
- لیست زنده «افراد/موارد داخل کارخانه» که با ثبت خروج، به‌روز می‌شود.
- ذخیره‌سازی دائمی با **Room (SQLite)** — تمام داده‌ها پس از بستن اپ باقی می‌مانند.
- طراحی متریال، پشتیبانی کامل از راست‌به‌چپ (RTL) و رابط کاربری فارسی.

## معماری فنی
```
app/src/main/java/ir/factory/entryexit/
├── data/
│   ├── PersonType.kt        // enum دسته‌ها: PERSONNEL, MACHINERY, VISITOR, DRIVER
│   ├── PersonEntity.kt      // جدول persons (شامل isInside)
│   ├── LogEntity.kt         // جدول logs (تاریخچه کامل ورود/خروج)
│   ├── PersonDao.kt / LogDao.kt
│   ├── AppDatabase.kt       // Room Database (Singleton)
│   └── Repository.kt        // قانون اصلی کسب‌وکار اینجا پیاده شده
├── viewmodel/
│   └── CategoryViewModel.kt
└── ui/
    ├── MainActivity.kt       // صفحه اصلی با ۴ دکمه
    ├── CategoryActivity.kt   // لیست + دیالوگ‌های ورود/خروج/افزودن
    └── PersonAdapter.kt      // آداپتور RecyclerView
```

منطق اصلی (جلوگیری از ورود تکراری) به‌طور کامل در `Repository.checkIn()` پیاده‌سازی شده:
هر فرد یک فیلد `isInside` دارد؛ اگر `true` باشد، `checkIn()` با شکست مواجه می‌شود و پیام مناسب نمایش داده می‌شود.
با ثبت خروج، `isInside` به `false` تغییر می‌کند و فرد از لیست «داخل کارخانه» حذف می‌شود.

## نحوه باز کردن و اجرای پروژه
1. [Android Studio](https://developer.android.com/studio) (نسخه Hedgehog یا بالاتر) را نصب کنید.
2. پوشه `ConcreteFactoryApp` را با گزینه **Open** در Android Studio باز کنید.
3. صبر کنید تا Gradle Sync به‌صورت خودکار انجام شود (اولین بار به اینترنت نیاز دارد تا وابستگی‌ها دانلود شوند).
4. یک دستگاه/شبیه‌ساز اندروید (حداقل API 24 / Android 7.0) انتخاب کرده و روی **Run ▶** کلیک کنید.

> نکته: فایل `gradle-wrapper.jar` به‌طور خودکار توسط Android Studio هنگام Sync ساخته می‌شود؛ نیازی به دانلود دستی نیست.

## ساخت فایل APK برای نصب مستقیم
در Android Studio:
`Build ▸ Build Bundle(s) / APK(s) ▸ Build APK(s)`
یا از خط فرمان (پس از باز کردن یک‌بار در Android Studio):
```
./gradlew assembleDebug
```
فایل خروجی در مسیر زیر قرار می‌گیرد:
```
app/build/outputs/apk/debug/app-debug.apk
```
